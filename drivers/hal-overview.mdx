The STM32H7 Hardware Abstraction Layer (HAL) is the primary driver layer in STM32CubeH7. It provides a uniform C API over every peripheral on every STM32H7 device — from GPIO and UART to Ethernet and JPEG — so your application code stays portable across the H743, H745, H750, H7B3 and other family members without touching peripheral register addresses directly.

## Layered architecture

```
┌─────────────────────────────────────────┐
│            Application code             │
├─────────────────────────────────────────┤
│        HAL drivers (stm32h7xx_hal_*.c)  │  ← portable, callback-based
├─────────────────────────────────────────┤
│        LL drivers (stm32h7xx_ll_*.h)    │  ← thin register-level inlines
├─────────────────────────────────────────┤
│  CMSIS Device (stm32h7xx.h, core_cm7.h) │  ← peripheral structs + NVIC
├─────────────────────────────────────────┤
│          Hardware (STM32H7xx)           │
└─────────────────────────────────────────┘
```

The HAL sits above the LL layer. Each HAL source file (`stm32h7xx_hal_uart.c`, `stm32h7xx_hal_spi.c`, …) is compiled independently; you include only the peripherals your application uses via the `stm32h7xx_hal_conf.h` configuration header.

## Initialisation flow

Every STM32H7 application follows the same three-step boot sequence before any peripheral can be used:

<Steps>
  <Step title="HAL_Init()">
    Resets all peripherals, initialises the Flash prefetch and instruction/data caches, configures the SysTick to 1 ms, and calls `HAL_MspInit()` where you perform any board-level initialisation.

    ```c
    HAL_Init();
    ```
  </Step>
  <Step title="SystemClock_Config()">
    Configures the PLL, selects the system clock source, and sets AHB/APB bus prescalers. You write this function — CubeMX generates it automatically.

    ```c
    void SystemClock_Config(void)
    {
        RCC_OscInitTypeDef       osc = {0};
        RCC_ClkInitTypeDef       clk = {0};

        /* Enable HSE oscillator */
        osc.OscillatorType = RCC_OSCILLATORTYPE_HSE;
        osc.HSEState       = RCC_HSE_ON;
        osc.PLL.PLLState   = RCC_PLL_ON;
        osc.PLL.PLLSource  = RCC_PLLSOURCE_HSE;
        osc.PLL.PLLM       = 5;
        osc.PLL.PLLN       = 192;
        osc.PLL.PLLP       = 2;   /* VCO / 2 = 480 MHz */
        osc.PLL.PLLQ       = 4;
        osc.PLL.PLLR       = 2;
        HAL_RCC_OscConfig(&osc);

        /* Select PLL as system clock; set bus dividers */
        clk.ClockType      = RCC_CLOCKTYPE_SYSCLK | RCC_CLOCKTYPE_HCLK |
                             RCC_CLOCKTYPE_PCLK1  | RCC_CLOCKTYPE_PCLK2;
        clk.SYSCLKSource   = RCC_SYSCLKSOURCE_PLLCLK;
        clk.AHBCLKDivider  = RCC_SYSCLK_DIV1;   /* HCLK = 480 MHz */
        clk.APB1CLKDivider = RCC_HCLK_DIV2;
        clk.APB2CLKDivider = RCC_HCLK_DIV2;
        HAL_RCC_ClockConfig(&clk, FLASH_LATENCY_4);
    }
    ```
  </Step>
  <Step title="Peripheral init (e.g., HAL_GPIO_Init)">
    Each peripheral has its own `Init` function that accepts a typed handle or init struct.

    ```c
    /* Enable GPIOB clock */
    __HAL_RCC_GPIOB_CLK_ENABLE();

    GPIO_InitTypeDef gpio = {0};
    gpio.Pin   = GPIO_PIN_0;
    gpio.Mode  = GPIO_MODE_OUTPUT_PP;
    gpio.Pull  = GPIO_NOPULL;
    gpio.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(GPIOB, &gpio);
    ```
  </Step>
</Steps>

## Handle structures

Every stateful HAL peripheral uses a `*_HandleTypeDef` structure as the unit of state. It holds the peripheral base address, configuration, DMA linkage, transfer buffers, and the current state machine value.

```c
/* UART handle — declare once, pass to every HAL_UART_* function */
UART_HandleTypeDef huart3;

huart3.Instance          = USART3;
huart3.Init.BaudRate     = 115200;
huart3.Init.WordLength   = UART_WORDLENGTH_8B;
huart3.Init.StopBits     = UART_STOPBITS_1;
huart3.Init.Parity       = UART_PARITY_NONE;
huart3.Init.Mode         = UART_MODE_TX_RX;
huart3.Init.HwFlowCtl    = UART_HWCONTROL_NONE;
huart3.Init.OverSampling = UART_OVERSAMPLING_16;
HAL_UART_Init(&huart3);
```

Other common handle types follow the same pattern:

| Handle | Peripheral |
|--------|-----------|
| `SPI_HandleTypeDef` | SPI1–SPI6 |
| `I2C_HandleTypeDef` | I2C1–I2C4 |
| `TIM_HandleTypeDef` | TIM1–TIM17 |
| `ADC_HandleTypeDef` | ADC1–ADC3 |
| `DMA_HandleTypeDef` | DMA1, DMA2, MDMA, BDMA |
| `ETH_HandleTypeDef` | Ethernet MAC |
| `QSPI_HandleTypeDef` | Quad-SPI |

## Transfer modes

HAL exposes three transfer modes for every peripheral that moves data:

<Tabs>
  <Tab title="Polling">
    The function blocks until the transfer completes or a timeout expires. Use for short, infrequent transfers where simplicity matters more than CPU utilisation.

    ```c
    uint8_t buf[] = "Hello\r\n";
    HAL_StatusTypeDef rc = HAL_UART_Transmit(&huart3, buf, sizeof(buf), 100 /* ms */);
    if (rc != HAL_OK) { /* handle error */ }
    ```
  </Tab>
  <Tab title="Interrupt">
    The function returns immediately; the NVIC fires an ISR on completion and the HAL calls a user-overridable callback.

    ```c
    /* Start non-blocking receive of 64 bytes */
    HAL_UART_Receive_IT(&huart3, rx_buf, 64U);

    /* Callback runs in IRQ context when all 64 bytes arrive */
    void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
    {
        if (huart->Instance == USART3)
        {
            /* process rx_buf */
        }
    }
    ```
  </Tab>
  <Tab title="DMA">
    The DMA controller moves data without CPU involvement. Use for large or continuous transfers (audio streaming, ADC sampling, bulk UART/SPI).

    ```c
    /* Link a configured DMA handle before calling the DMA variant */
    huart3.hdmatx = &hdma_usart3_tx;

    HAL_UART_Transmit_DMA(&huart3, tx_buf, TX_LEN);

    /* Called from DMA IRQ handler when transfer finishes */
    void HAL_UART_TxCpltCallback(UART_HandleTypeDef *huart)
    {
        /* DMA transfer complete */
    }
    ```
  </Tab>
</Tabs>

## HAL_StatusTypeDef

Every HAL function returns one of four values:

| Value | Meaning |
|-------|---------|
| `HAL_OK` (0) | Operation succeeded |
| `HAL_ERROR` (1) | A hardware or parameter error occurred |
| `HAL_BUSY` (2) | The peripheral is already in use |
| `HAL_TIMEOUT` (3) | The operation did not complete within the allotted time |

```c
HAL_StatusTypeDef status = HAL_SPI_Transmit(&hspi1, data, len, 10U);

switch (status)
{
    case HAL_OK:      break;
    case HAL_TIMEOUT: Error_Handler(); break;
    case HAL_BUSY:    /* retry later */ break;
    case HAL_ERROR:   Error_Handler(); break;
}
```

## Callbacks

The HAL uses weak-linked callback functions so you override only the ones you need. The compiler picks up your definition automatically — no registration required.

```c
/* Override the half-transfer callback for double-buffered DMA audio */
void HAL_SAI_RxHalfCpltCallback(SAI_HandleTypeDef *hsai)
{
    /* Process first half of DMA buffer while second half fills */
    AudioProcess(audio_buf, AUDIO_BUF_SIZE / 2U);
}

void HAL_SAI_RxCpltCallback(SAI_HandleTypeDef *hsai)
{
    /* Process second half */
    AudioProcess(audio_buf + AUDIO_BUF_SIZE / 2U, AUDIO_BUF_SIZE / 2U);
}
```

## MSP callbacks

`HAL_*_MspInit()` and `HAL_*_MspDeInit()` are the points where you enable clocks, configure GPIO alternate functions, and set up DMA or NVIC — everything that is board-specific rather than peripheral-logic-specific.

```c
void HAL_UART_MspInit(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART3)
    {
        __HAL_RCC_USART3_CLK_ENABLE();
        __HAL_RCC_GPIOD_CLK_ENABLE();

        /* PD8 = USART3_TX, PD9 = USART3_RX (AF7) */
        GPIO_InitTypeDef gpio = {0};
        gpio.Pin       = GPIO_PIN_8 | GPIO_PIN_9;
        gpio.Mode      = GPIO_MODE_AF_PP;
        gpio.Pull      = GPIO_NOPULL;
        gpio.Speed     = GPIO_SPEED_FREQ_VERY_HIGH;
        gpio.Alternate = GPIO_AF7_USART3;
        HAL_GPIO_Init(GPIOD, &gpio);
    }
}
```

## Configuration header

`stm32h7xx_hal_conf.h` controls which peripheral modules are compiled in and sets global HAL options:

```c
/* Enable only the modules you use */
#define HAL_MODULE_ENABLED
#define HAL_GPIO_MODULE_ENABLED
#define HAL_UART_MODULE_ENABLED
#define HAL_SPI_MODULE_ENABLED
#define HAL_DMA_MODULE_ENABLED
#define HAL_RCC_MODULE_ENABLED
#define HAL_CORTEX_MODULE_ENABLED
#define HAL_FLASH_MODULE_ENABLED
#define HAL_PWR_MODULE_ENABLED

/* External oscillator frequency (Hz) */
#define HSE_VALUE  25000000U

/* SysTick-based HAL timeout tick (ms) */
#define HAL_GetTick()  HAL_GetTick()
```

<Warning>
  Never call HAL functions from inside an ISR whose priority is higher than `configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY` when using FreeRTOS — the HAL's internal timeout polling uses `HAL_GetTick()` which relies on SysTick, and re-entering the scheduler from a high-priority ISR will hard-fault the system.
</Warning>
