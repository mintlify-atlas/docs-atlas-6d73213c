title: "HAL RTC driver — real-time clock and calendar"
description: "Configure the STM32H7 RTC for timekeeping, calendar management, and alarm generation using LSE or LSI as the independent clock source."
---

The STM32H7 RTC is an independent BCD calendar with sub-second resolution, two programmable alarms (A and B), a periodic wakeup timer, and a tamper/timestamp unit. It runs from a dedicated clock domain — typically the 32.768 kHz LSE crystal — that remains active in Standby and VBAT modes, preserving time and date across system resets and power cycles as long as VBAT is supplied.

<Note>
The RTC domain is under the backup power supply (VBAT). On a first run after VBAT power-on, you must enable access to the backup domain via `HAL_PWR_EnableBkUpAccess()` and reset the backup domain with `__HAL_RCC_BACKUPRESET_FORCE()` / `__HAL_RCC_BACKUPRESET_RELEASE()` before calling `HAL_RTC_Init()`.
</Note>

## Clock source

Select the RTC clock source in the RCC backup domain configuration before calling `HAL_RTC_Init()`.

| Source | Frequency | Notes |
|--------|-----------|-------|
| LSE (recommended) | 32.768 kHz | External crystal; survives Standby/VBAT; best accuracy |
| LSI | ~32 kHz | Internal RC; no external component; ±5% accuracy |
| HSE/32 | HSE/32 | Not available in Standby |

With LSE at 32.768 kHz, the standard prescaler values are `AsynchPrediv = 127` and `SynchPrediv = 255`, yielding a 1 Hz calendar clock with 1/256 s sub-second resolution.

## Key types

### RTC_InitTypeDef

<ParamField path="body.HourFormat" type="uint32_t" required>
  Hour representation. `RTC_HOURFORMAT_24` for 24-hour format or `RTC_HOURFORMAT_12` for 12-hour AM/PM format.
</ParamField>

<ParamField path="body.AsynchPrediv" type="uint32_t" required>
  Asynchronous prescaler value (7-bit, range 0–127). Divides the RTC input clock before the synchronous prescaler. The asynchronous prescaler should be as high as possible to minimize power in stop modes.
</ParamField>

<ParamField path="body.SynchPrediv" type="uint32_t" required>
  Synchronous prescaler value (15-bit, range 0–32767). With LSE, set to `255` to achieve a 1 Hz calendar clock.
</ParamField>

<ParamField path="body.OutPut" type="uint32_t" required>
  Selects the signal routed to the RTC_AF1 output pin. `RTC_OUTPUT_DISABLE`, `RTC_OUTPUT_ALARMA`, `RTC_OUTPUT_ALARMB`, or `RTC_OUTPUT_WAKEUP`.
</ParamField>

<ParamField path="body.OutPutRemap" type="uint32_t">
  Remaps the RTC output to the alternate pin. `RTC_OUTPUT_REMAP_NONE` or `RTC_OUTPUT_REMAP_POS1`.
</ParamField>

<ParamField path="body.OutPutPolarity" type="uint32_t" required>
  Output polarity. `RTC_OUTPUT_POLARITY_HIGH` or `RTC_OUTPUT_POLARITY_LOW`.
</ParamField>

<ParamField path="body.OutPutType" type="uint32_t" required>
  Output driver type. `RTC_OUTPUT_TYPE_OPENDRAIN` or `RTC_OUTPUT_TYPE_PUSHPULL`.
</ParamField>

### RTC_TimeTypeDef

<ParamField path="body.Hours" type="uint8_t" required>
  Hours value. Range 0–23 in 24-hour format, or 1–12 in 12-hour format. Pass in BCD (`RTC_FORMAT_BCD`) or binary (`RTC_FORMAT_BIN`) according to the `Format` argument of `HAL_RTC_SetTime()`.
</ParamField>

<ParamField path="body.Minutes" type="uint8_t" required>
  Minutes value (0–59).
</ParamField>

<ParamField path="body.Seconds" type="uint8_t" required>
  Seconds value (0–59).
</ParamField>

<ParamField path="body.SubSeconds" type="uint32_t">
  Sub-second register value (read-only on `HAL_RTC_GetTime()`). Counts down from `SynchPrediv`. Resolution is `1 / (SynchPrediv + 1)` seconds.
</ParamField>

<ParamField path="body.TimeFormat" type="uint8_t">
  AM/PM indicator. `RTC_HOURFORMAT12_AM` or `RTC_HOURFORMAT12_PM`. Ignored in 24-hour format.
</ParamField>

<ParamField path="body.DayLightSaving" type="uint32_t">
  One-time daylight saving adjustment. `RTC_DAYLIGHTSAVING_ADD1H`, `RTC_DAYLIGHTSAVING_SUB1H`, or `RTC_DAYLIGHTSAVING_NONE`.
</ParamField>

<ParamField path="body.StoreOperation" type="uint32_t">
  Records a daylight saving flag in the BKP register. `RTC_STOREOPERATION_SET` or `RTC_STOREOPERATION_RESET`.
</ParamField>

### RTC_DateTypeDef

<ParamField path="body.WeekDay" type="uint8_t" required>
  Day of the week. `RTC_WEEKDAY_MONDAY` through `RTC_WEEKDAY_SUNDAY`.
</ParamField>

<ParamField path="body.Month" type="uint8_t" required>
  Month. `RTC_MONTH_JANUARY` through `RTC_MONTH_DECEMBER`.
</ParamField>

<ParamField path="body.Date" type="uint8_t" required>
  Day of the month (1–31).
</ParamField>

<ParamField path="body.Year" type="uint8_t" required>
  Year offset from 2000 (0–99). The value `0x25` represents 2025.
</ParamField>

## Key functions

### HAL_RTC_Init

```c
HAL_StatusTypeDef HAL_RTC_Init(RTC_HandleTypeDef *hrtc);
```

Configures the RTC prescalers, hour format, and output settings. If the RTC is already running (backup domain was not reset), this function skips re-initialization to preserve the current time and date.

### HAL_RTC_SetTime / HAL_RTC_GetTime

```c
HAL_StatusTypeDef HAL_RTC_SetTime(RTC_HandleTypeDef *hrtc,
                                   RTC_TimeTypeDef *sTime,
                                   uint32_t Format);

HAL_StatusTypeDef HAL_RTC_GetTime(RTC_HandleTypeDef *hrtc,
                                   RTC_TimeTypeDef *sTime,
                                   uint32_t Format);
```

`Format` is `RTC_FORMAT_BIN` (binary integers) or `RTC_FORMAT_BCD` (packed BCD).

<Warning>
Always call `HAL_RTC_GetTime()` before `HAL_RTC_GetDate()`. The shadow registers are locked until the date register is read. If you read only the time, the `SubSeconds` value is not guaranteed to be coherent with the seconds counter.
</Warning>

### HAL_RTC_SetDate / HAL_RTC_GetDate

```c
HAL_StatusTypeDef HAL_RTC_SetDate(RTC_HandleTypeDef *hrtc,
                                   RTC_DateTypeDef *sDate,
                                   uint32_t Format);

HAL_StatusTypeDef HAL_RTC_GetDate(RTC_HandleTypeDef *hrtc,
                                   RTC_DateTypeDef *sDate,
                                   uint32_t Format);
```

### HAL_RTC_SetAlarm / HAL_RTC_SetAlarm_IT

```c
/* Polling mode */
HAL_StatusTypeDef HAL_RTC_SetAlarm(RTC_HandleTypeDef *hrtc,
                                    RTC_AlarmTypeDef *sAlarm,
                                    uint32_t Format);

/* Interrupt mode */
HAL_StatusTypeDef HAL_RTC_SetAlarm_IT(RTC_HandleTypeDef *hrtc,
                                       RTC_AlarmTypeDef *sAlarm,
                                       uint32_t Format);
```

Configures Alarm A or Alarm B. Use `HAL_RTC_SetAlarm_IT()` to generate an interrupt when the alarm matches. The `AlarmMask` field controls which fields are compared: set bits to `RTC_ALARMMASK_*` constants to ignore that field.

### HAL_RTC_AlarmIRQHandler

```c
void HAL_RTC_AlarmIRQHandler(RTC_HandleTypeDef *hrtc);
```

Call from the `RTC_Alarm_IRQHandler` interrupt handler in `stm32h7xx_it.c`. Clears the alarm flag and dispatches the event to the appropriate callback.

### HAL_RTC_AlarmAEventCallback

```c
void HAL_RTC_AlarmAEventCallback(RTC_HandleTypeDef *hrtc);
```

Weak callback invoked by `HAL_RTC_AlarmIRQHandler()` when Alarm A fires. Override this in application code to implement alarm handling.

## Time set and get example

```c
RTC_HandleTypeDef hrtc;
RTC_TimeTypeDef   sTime;
RTC_DateTypeDef   sDate;

/* --- Initialization --- */
hrtc.Instance            = RTC;
hrtc.Init.HourFormat     = RTC_HOURFORMAT_24;
hrtc.Init.AsynchPrediv   = 127;   /* (128-1): divides 32768 Hz to 256 Hz */
hrtc.Init.SynchPrediv    = 255;   /* (256-1): divides 256 Hz to 1 Hz    */
hrtc.Init.OutPut         = RTC_OUTPUT_DISABLE;
hrtc.Init.OutPutPolarity = RTC_OUTPUT_POLARITY_HIGH;
hrtc.Init.OutPutType     = RTC_OUTPUT_TYPE_OPENDRAIN;

if (HAL_RTC_Init(&hrtc) != HAL_OK)
{
    Error_Handler();
}

/* --- Set date: Tuesday, 29 April 2025 --- */
sDate.WeekDay = RTC_WEEKDAY_TUESDAY;
sDate.Month   = RTC_MONTH_APRIL;
sDate.Date    = 29;
sDate.Year    = 25;  /* offset from 2000 */

HAL_RTC_SetDate(&hrtc, &sDate, RTC_FORMAT_BIN);

/* --- Set time: 14:30:00 --- */
sTime.Hours          = 14;
sTime.Minutes        = 30;
sTime.Seconds        = 0;
sTime.DayLightSaving = RTC_DAYLIGHTSAVING_NONE;
sTime.StoreOperation = RTC_STOREOPERATION_RESET;

HAL_RTC_SetTime(&hrtc, &sTime, RTC_FORMAT_BIN);

/* --- Read back time and date --- */
RTC_TimeTypeDef currentTime;
RTC_DateTypeDef currentDate;

/* Always read time first, then date */
HAL_RTC_GetTime(&hrtc, &currentTime, RTC_FORMAT_BIN);
HAL_RTC_GetDate(&hrtc, &currentDate, RTC_FORMAT_BIN);
```

## Alarm configuration example

```c
RTC_AlarmTypeDef sAlarm;

/* Fire Alarm A at 14:30:30, ignoring the date/weekday field */
sAlarm.Alarm                     = RTC_ALARM_A;
sAlarm.AlarmDateWeekDaySel       = RTC_ALARMDATEWEEKDAYSEL_WEEKDAY;
sAlarm.AlarmDateWeekDay          = RTC_WEEKDAY_TUESDAY;
sAlarm.AlarmMask                 = RTC_ALARMMASK_DATEWEEKDAY; /* ignore date */
sAlarm.AlarmSubSecondMask        = RTC_ALARMSUBSECONDMASK_NONE;
sAlarm.AlarmTime.TimeFormat      = RTC_HOURFORMAT12_AM;
sAlarm.AlarmTime.Hours           = 14;
sAlarm.AlarmTime.Minutes         = 30;
sAlarm.AlarmTime.Seconds         = 30;
sAlarm.AlarmTime.SubSeconds      = 0;

if (HAL_RTC_SetAlarm_IT(&hrtc, &sAlarm, RTC_FORMAT_BIN) != HAL_OK)
{
    Error_Handler();
}
```

In `stm32h7xx_it.c`, forward the interrupt to the HAL handler:

```c
void RTC_Alarm_IRQHandler(void)
{
    HAL_RTC_AlarmIRQHandler(&hrtc);
}
```

Override the callback in application code:

```c
void HAL_RTC_AlarmAEventCallback(RTC_HandleTypeDef *hrtc)
{
    /* Alarm A fired — toggle an LED, log an event, etc. */
    BSP_LED_Toggle(LED1);
}
```

## Backup registers

The RTC provides 32 × 32-bit backup registers (`RTC_BKPxR`) that retain their values in Standby and VBAT modes. Use them to store the initialization state flag so that the RTC is not re-initialized on every reset:

```c
/* On first boot, write a magic value after RTC init */
HAL_RTCEx_BKUPWrite(&hrtc, RTC_BKP_DR0, 0x32F2U);

/* On subsequent boots, check the flag before re-initializing */
if (HAL_RTCEx_BKUPRead(&hrtc, RTC_BKP_DR0) != 0x32F2U)
{
    /* RTC not initialized — run full init sequence */
}
```
