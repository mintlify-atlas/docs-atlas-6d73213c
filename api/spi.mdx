title: "HAL SPI driver — synchronous serial interface"
description: "STM32H7 HAL SPI driver reference: SPI_InitTypeDef fields, polling, interrupt, and DMA transfer functions, and SPI1–SPI6 peripheral capabilities."
---

The STM32H7 HAL SPI driver supports full-duplex, half-duplex, and simplex communication in master and slave configurations. STM32H7 devices include up to six SPI instances: SPI1–SPI3 are clocked from the APB2/APB1 bus (up to 100 MHz), while SPI4–SPI6 run from the APB2 bus and can reach up to 150 MHz on devices with the high-speed clock. All instances share the same `SPI_HandleTypeDef` handle and API.

<Note>
The STM32H7 SPI peripheral is a redesigned IP compared to STM32F4/F7. It introduces a FIFO, separate data-size and FIFO-threshold controls, and the `MasterKeepIOState` field to prevent glitches when NSS is software-controlled.
</Note>

## SPI_InitTypeDef

Populate `SPI_HandleTypeDef.Init` before calling `HAL_SPI_Init`.

<ParamField path="Init.Mode" type="uint32_t" required>
  Selects master or slave operation: `SPI_MODE_MASTER` or `SPI_MODE_SLAVE`.
</ParamField>

<ParamField path="Init.Direction" type="uint32_t" required>
  Communication direction: `SPI_DIRECTION_2LINES` (full-duplex), `SPI_DIRECTION_2LINES_RXONLY`, or `SPI_DIRECTION_1LINE` (half-duplex bidirectional).
</ParamField>

<ParamField path="Init.DataSize" type="uint32_t" required>
  Frame width from 4 to 32 bits. Common values: `SPI_DATASIZE_8BIT`, `SPI_DATASIZE_16BIT`, `SPI_DATASIZE_32BIT`.
</ParamField>

<ParamField path="Init.CLKPolarity" type="uint32_t" required>
  Clock idle state: `SPI_POLARITY_LOW` (CPOL=0) or `SPI_POLARITY_HIGH` (CPOL=1).
</ParamField>

<ParamField path="Init.CLKPhase" type="uint32_t" required>
  Data capture edge: `SPI_PHASE_1EDGE` (CPHA=0, leading edge) or `SPI_PHASE_2EDGE` (CPHA=1, trailing edge).
</ParamField>

<ParamField path="Init.NSS" type="uint32_t" required>
  NSS management: `SPI_NSS_SOFT` (software-controlled via `SPI_CR1.SSI`), `SPI_NSS_HARD_INPUT`, or `SPI_NSS_HARD_OUTPUT`.
</ParamField>

<ParamField path="Init.BaudRatePrescaler" type="uint32_t" required>
  Clock divider applied to the peripheral clock: `SPI_BAUDRATEPRESCALER_2` through `SPI_BAUDRATEPRESCALER_256` in powers of two.
</ParamField>

<ParamField path="Init.FirstBit" type="uint32_t" required>
  Bit order: `SPI_FIRSTBIT_MSB` (default) or `SPI_FIRSTBIT_LSB`.
</ParamField>

<ParamField path="Init.TIMode" type="uint32_t" required>
  Texas Instruments synchronous serial mode: `SPI_TIMODE_DISABLE` (standard Motorola SPI) or `SPI_TIMODE_ENABLE`.
</ParamField>

<ParamField path="Init.CRCCalculation" type="uint32_t" required>
  Hardware CRC: `SPI_CRCCALCULATION_DISABLE` or `SPI_CRCCALCULATION_ENABLE`. When enabled, also set `CRCPolynomial` and `CRCLength`.
</ParamField>

## Initialization

### HAL_SPI_Init

```c
HAL_StatusTypeDef HAL_SPI_Init(SPI_HandleTypeDef *hspi);
```

Configures the SPI peripheral, calls `HAL_SPI_MspInit` for GPIO and clock setup, and places the peripheral in the ready state.

### HAL_SPI_DeInit

```c
HAL_StatusTypeDef HAL_SPI_DeInit(SPI_HandleTypeDef *hspi);
```

Disables the peripheral and calls `HAL_SPI_MspDeInit`. Call before changing configuration.

**Master init example (SPI1, Mode 0, 8-bit, software NSS):**

```c
SPI_HandleTypeDef hspi1;

hspi1.Instance               = SPI1;
hspi1.Init.Mode              = SPI_MODE_MASTER;
hspi1.Init.Direction         = SPI_DIRECTION_2LINES;
hspi1.Init.DataSize          = SPI_DATASIZE_8BIT;
hspi1.Init.CLKPolarity       = SPI_POLARITY_LOW;
hspi1.Init.CLKPhase          = SPI_PHASE_1EDGE;
hspi1.Init.NSS               = SPI_NSS_SOFT;
hspi1.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_16;
hspi1.Init.FirstBit          = SPI_FIRSTBIT_MSB;
hspi1.Init.TIMode            = SPI_TIMODE_DISABLE;
hspi1.Init.CRCCalculation    = SPI_CRCCALCULATION_DISABLE;
hspi1.Init.CRCPolynomial     = 7;
hspi1.Init.CRCLength         = SPI_CRC_LENGTH_8BIT;
hspi1.Init.NSSPMode          = SPI_NSS_PULSE_DISABLE;
hspi1.Init.MasterKeepIOState = SPI_MASTER_KEEP_IO_STATE_ENABLE;

HAL_SPI_Init(&hspi1);
```

<Tip>
Set `MasterKeepIOState = SPI_MASTER_KEEP_IO_STATE_ENABLE` when using software NSS. This prevents the MOSI and SCK lines from floating between transactions.
</Tip>

## Transfer functions

<Tabs>
  <Tab title="Polling">
    Polling transfers block the CPU until all bytes are exchanged or the timeout expires.

    ### HAL_SPI_Transmit

    ```c
    HAL_StatusTypeDef HAL_SPI_Transmit(SPI_HandleTypeDef *hspi,
                                       const uint8_t *pData,
                                       uint16_t Size,
                                       uint32_t Timeout);
    ```

    Sends `Size` bytes. Received data is discarded (RX FIFO is flushed).

    ### HAL_SPI_Receive

    ```c
    HAL_StatusTypeDef HAL_SPI_Receive(SPI_HandleTypeDef *hspi,
                                      uint8_t *pData,
                                      uint16_t Size,
                                      uint32_t Timeout);
    ```

    Clocks in `Size` bytes. Dummy bytes are sent on MOSI.

    ### HAL_SPI_TransmitReceive

    ```c
    HAL_StatusTypeDef HAL_SPI_TransmitReceive(SPI_HandleTypeDef *hspi,
                                              const uint8_t *pTxData,
                                              uint8_t *pRxData,
                                              uint16_t Size,
                                              uint32_t Timeout);
    ```

    Full-duplex exchange: simultaneously sends from `pTxData` and stores received bytes to `pRxData`.

    ```c
    uint8_t txBuf[] = {0x9F, 0x00, 0x00, 0x00}; /* Read JEDEC ID command */
    uint8_t rxBuf[4];

    HAL_GPIO_WritePin(CS_GPIO_Port, CS_Pin, GPIO_PIN_RESET);
    HAL_SPI_TransmitReceive(&hspi1, txBuf, rxBuf, sizeof(txBuf), 1000);
    HAL_GPIO_WritePin(CS_GPIO_Port, CS_Pin, GPIO_PIN_SET);
    /* rxBuf[1..3] contains the JEDEC manufacturer/device ID */
    ```
  </Tab>

  <Tab title="Interrupt">
    Interrupt-driven transfers return immediately. Set `SPI_HandleTypeDef` callbacks and call `HAL_SPI_IRQHandler` from the peripheral's IRQ handler.

    ### HAL_SPI_Transmit_IT

    ```c
    HAL_StatusTypeDef HAL_SPI_Transmit_IT(SPI_HandleTypeDef *hspi,
                                          const uint8_t *pData,
                                          uint16_t Size);
    ```

    ### HAL_SPI_Receive_IT

    ```c
    HAL_StatusTypeDef HAL_SPI_Receive_IT(SPI_HandleTypeDef *hspi,
                                         uint8_t *pData,
                                         uint16_t Size);
    ```

    ### HAL_SPI_TransmitReceive_IT

    ```c
    HAL_StatusTypeDef HAL_SPI_TransmitReceive_IT(SPI_HandleTypeDef *hspi,
                                                 const uint8_t *pTxData,
                                                 uint8_t *pRxData,
                                                 uint16_t Size);
    ```

    ```c
    /* In stm32h7xx_it.c */
    void SPI1_IRQHandler(void)
    {
      HAL_SPI_IRQHandler(&hspi1);
    }

    /* Start non-blocking full-duplex transfer */
    uint8_t txBuf[16], rxBuf[16];
    HAL_SPI_TransmitReceive_IT(&hspi1, txBuf, rxBuf, sizeof(txBuf));

    /* Callback fires on completion */
    void HAL_SPI_TxRxCpltCallback(SPI_HandleTypeDef *hspi)
    {
      if (hspi->Instance == SPI1)
      {
        /* Process rxBuf */
      }
    }
    ```
  </Tab>

  <Tab title="DMA">
    DMA transfers free the CPU entirely during the transfer. Suitable for large buffers or continuous streaming.

    ### HAL_SPI_Transmit_DMA

    ```c
    HAL_StatusTypeDef HAL_SPI_Transmit_DMA(SPI_HandleTypeDef *hspi,
                                           const uint8_t *pData,
                                           uint16_t Size);
    ```

    ### HAL_SPI_Receive_DMA

    ```c
    HAL_StatusTypeDef HAL_SPI_Receive_DMA(SPI_HandleTypeDef *hspi,
                                          uint8_t *pData,
                                          uint16_t Size);
    ```

    ### HAL_SPI_TransmitReceive_DMA

    ```c
    HAL_StatusTypeDef HAL_SPI_TransmitReceive_DMA(SPI_HandleTypeDef *hspi,
                                                   const uint8_t *pTxData,
                                                   uint8_t *pRxData,
                                                   uint16_t Size);
    ```

    <Warning>
    SPI DMA buffers must be located in AXI SRAM or SRAM1/2 — not DTCM or ITCM. Cache coherency must also be managed: either use cache-invalidate/clean operations (`SCB_CleanDCache_by_Addr` / `SCB_InvalidateDCache_by_Addr`) or place buffers in non-cacheable memory.
    </Warning>

    ```c
    /* Buffer in DMA-accessible, non-cacheable memory */
    #if defined ( __GNUC__ )
    __attribute__((section(".dma_buffer")))
    #endif
    uint8_t txBuf[256], rxBuf[256];

    HAL_SPI_TransmitReceive_DMA(&hspi1, txBuf, rxBuf, sizeof(txBuf));

    void HAL_SPI_TxRxCpltCallback(SPI_HandleTypeDef *hspi)
    {
      /* Clean/invalidate cache if needed before reading rxBuf */
      SCB_InvalidateDCache_by_Addr((uint32_t *)rxBuf, sizeof(rxBuf));
      /* Process data */
    }
    ```
  </Tab>
</Tabs>

## Callbacks

| Function | Trigger |
|---|---|
| `HAL_SPI_TxCpltCallback` | Transmit-only transfer complete |
| `HAL_SPI_RxCpltCallback` | Receive-only transfer complete |
| `HAL_SPI_TxRxCpltCallback` | Full-duplex transfer complete |
| `HAL_SPI_TxHalfCpltCallback` | Half of TX buffer sent (DMA) |
| `HAL_SPI_RxHalfCpltCallback` | Half of RX buffer received (DMA) |
| `HAL_SPI_TxRxHalfCpltCallback` | Half of full-duplex transfer done (DMA) |
| `HAL_SPI_ErrorCallback` | Error during any transfer |

Inspect `hspi->ErrorCode` inside `HAL_SPI_ErrorCallback` for the specific fault (`HAL_SPI_ERROR_OVR`, `HAL_SPI_ERROR_MODF`, `HAL_SPI_ERROR_DMA`, etc.).

## SPI instances on STM32H7

| Instance | Max clock | Bus | Notes |
|---|---|---|---|
| SPI1 | 100 MHz | APB2 | |
| SPI2 | 50 MHz | APB1 | |
| SPI3 | 50 MHz | APB1 | |
| SPI4 | 100 MHz | APB2 | |
| SPI5 | 100 MHz | APB2 | |
| SPI6 | 100 MHz | APB4 | Can remain active in Stop mode |

<Note>
On STM32H7 devices with the 480 MHz operating point (e.g., STM32H743 rev V), APB2 can reach 120 MHz, allowing SPI4–SPI5 to exceed 100 MHz when using `SPI_BAUDRATEPRESCALER_2`.
</Note>
