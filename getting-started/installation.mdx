STM32CubeH7 is distributed as a Git repository that uses submodules to track the underlying HAL drivers, CMSIS headers, and middleware components. Cloning it correctly — with submodule initialization — is the single most common setup mistake, so this page walks you through every variant in order.

## Prerequisites

You need the following before you start:

- **Git 2.13 or later** — submodule support is required
- One of the supported toolchains listed below
- A NUCLEO-H743ZI or another supported STM32H7 board (optional for browsing examples)

<CardGroup cols={2}>
  <Card title="STM32CubeIDE" icon="microchip">
    The official free IDE from STMicroelectronics. Includes GCC ARM, GDB, and the ST-LINK flashing tools bundled together.
  </Card>
  <Card title="Keil MDK-ARM" icon="wrench">
    Commercial IDE. Project files are located in the `MDK-ARM/` sub-directory of each example.
  </Card>
  <Card title="IAR EWARM" icon="wrench">
    Commercial IDE. Project files are located in the `EWARM/` sub-directory of each example.
  </Card>
  <Card title="GCC ARM (bare toolchain)" icon="terminal">
    Use any editor with the `arm-none-eabi-gcc` toolchain. Linker scripts (`*.ld`) are provided in each `STM32CubeIDE/` directory.
  </Card>
</CardGroup>

## Clone the repository

The repository was created with `git submodule`. Cloning without `--recursive` leaves all driver directories empty.

<CodeGroup>

```bash Clone with submodules (recommended)
git clone --recursive https://github.com/STMicroelectronics/STM32CubeH7.git
```

```bash Clone a specific version directly
# Replace vX.Y.Z with the target version, e.g. v1.13.0
git clone --recursive --depth 1 --branch vX.Y.Z https://github.com/STMicroelectronics/STM32CubeH7.git
```

</CodeGroup>

<Warning>
If you use GitHub's **Download ZIP** option instead of `git clone`, the submodules are not included. You must collect and add them manually. Use `git clone --recursive` to avoid this.
</Warning>

## Update an existing clone

If you already have the repository on your machine, pull the latest upstream changes and then sync the submodules in one sequence:

```bash
git pull
git submodule update --init --recursive
```

Both commands are required. `git pull` updates only the top-level repo; `git submodule update` brings the driver submodules into sync with the new commit pointers.

## Pin to a specific version

The GitHub repository uses a rolling release and may be ahead of the version available on st.com or generated by STM32CubeMX. To pin your working copy to a known-good release tag after cloning:

```bash
# After cloning, check out the desired tag
git checkout v1.13.0
```

<Note>
The `--depth 1` flag shown in the direct-version clone above is optional. It skips downloading the full commit history up to the targeted version, which reduces download size but prevents you from running `git log` across older commits.
</Note>

## Verify the clone

After a successful clone, the `Drivers/` directory must be populated:

```bash
ls Drivers/
# Expected output:
# BSP/  CMSIS/  STM32H7xx_HAL_Driver/
```

If `Drivers/STM32H7xx_HAL_Driver/` is empty, the submodule initialization did not complete. Run:

```bash
git submodule update --init --recursive
```

## Repository layout

```
STM32CubeH7/
├── Drivers/
│   ├── BSP/                  # Board Support Package drivers
│   ├── CMSIS/                # ARM CMSIS core + STM32H7 device headers
│   └── STM32H7xx_HAL_Driver/ # HAL and LL driver source
├── Middlewares/
│   ├── ST/                   # ST middleware (FreeRTOS port, USB, FatFS, …)
│   └── Third_Party/          # FreeRTOS, FatFS, LwIP source
├── Projects/
│   ├── NUCLEO-H743ZI/        # Examples, Applications, Templates
│   ├── NUCLEO-H743ZI2/
│   └── …                     # One directory per supported board
└── Utilities/
```

## Unavailable middleware libraries

Three middleware components are **not included** in this GitHub repository because they require acceptance of a separate license via a click-through procedure on st.com:

| Library | Path in full package |
|---|---|
| STM32\_Audio | `Middlewares/ST/STM32_Audio` |
| STemWin | `Middlewares/ST/STemWin` |
| TouchGFX | `Middlewares/ST/TouchGFX` |

Any example or application that depends on these libraries is also absent from the repository. Download the full firmware package from [st.com](https://www.st.com/en/embedded-software/stm32cubeh7.html#get-software) if you need them. You will be prompted to log in or register.

<Tip>
If you use STM32CubeMX to generate a project that includes TouchGFX or STemWin, CubeMX will download the required middleware automatically during code generation — you do not need to clone the full package manually in that workflow.
</Tip>

## Toolchain-specific notes

<Tabs>
  <Tab title="STM32CubeIDE">
    Open a project by selecting **File → Import → Existing Projects into Workspace** and pointing to the `STM32CubeIDE/` sub-directory inside any example folder. The `.project` and `.cproject` files are already configured with the correct include paths and linker scripts.

    Two linker scripts are provided for the NUCLEO-H743ZI template:

    - `STM32H743ZITx_FLASH_DTCMRAM.ld` — places the default RAM region in DTCM RAM
    - `STM32H743ZITx_FLASH_RAM_D1.ld` — places the default RAM region in D1 AXI-SRAM
  </Tab>
  <Tab title="Keil MDK-ARM">
    Open the `.uvprojx` file inside the `MDK-ARM/` sub-directory. To switch the default RAM region, go to **Project options → Target** and toggle between IRAM1 (DTCM RAM) and IRAM2 (D1 AXI-SRAM).
  </Tab>
  <Tab title="IAR EWARM">
    Open the `.eww` workspace file inside the `EWARM/` sub-directory. To switch the default RAM region, go to **Project options → Linker → Config** and select either `stm32h743xx_flash_dtcmram.icf` or `stm32h743xx_flash_ram_d1.icf`.
  </Tab>
  <Tab title="GCC ARM (bare)">
    Use the linker scripts from the `STM32CubeIDE/` sub-directory directly. A typical GCC invocation for the NUCLEO-H743ZI template uses:

    ```bash
    arm-none-eabi-gcc -mcpu=cortex-m7 -mfpu=fpv5-d16 -mfloat-abi=hard \
      -DSTM32H743xx -DUSE_HAL_DRIVER \
      -T STM32H743ZITx_FLASH_DTCMRAM.ld \
      -o firmware.elf Src/*.c
    ```
  </Tab>
</Tabs>
