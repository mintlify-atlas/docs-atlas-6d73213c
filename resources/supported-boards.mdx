STM32CubeH7 v1.13.0 ships with validated BSP drivers and project examples for 21 official ST boards spanning the STM32H7 portfolio. Each board directory under `Projects/` contains Examples, Applications, Demonstrations, and Templates targeting that specific hardware.

  NUCLEO-H745ZI-Q and NUCLEO-H755ZI-Q carry dual-core MCUs. Projects for these boards include separate Cortex-M7 and Cortex-M4 firmware images that run concurrently using the hardware semaphore (HSEM) peripheral for inter-processor communication.
</Note>

## Dual-core boards summary

Four MCU families in this portfolio implement a Cortex-M7 + Cortex-M4 heterogeneous dual-core architecture. The table below lists every supported board that carries one of these MCUs.

| MCU | Boards |
|---|---|
| STM32H745 | NUCLEO-H745ZI-Q, STM32H745I-DISCO |
| STM32H747 | STM32H747I-DISCO, STM32H747I-DISC1, STM32H747I-EVAL |
| STM32H755 | NUCLEO-H755ZI-Q |
| STM32H757 | STM32H757I-EVAL |

<Warning>
  Dual-core projects require you to build and flash two separate ELF images — one for each core. The Cortex-M7 image is flashed to Bank 1 of internal Flash, and the Cortex-M4 image to Bank 2. Flashing only one image will leave the second core in reset.
</Warning>
