The `Examples/` and `Examples_LL/` directories contain single-peripheral, single-concept projects designed to demonstrate one HAL or LL feature in isolation. Each example is self-contained: it includes its own `main.c`, startup file, linker scripts, and IDE project files. They are the fastest way to verify hardware bring-up or understand a specific peripheral's API.

## GPIO blink example walkthrough

`Projects/NUCLEO-H743ZI/Examples/GPIO/GPIO_Demo` is a good first project to open. It demonstrates LED toggle speed control driven by a user button press through EXTI.

```c
/* GPIO/GPIO_Demo/Src/main.c — main loop */
#define LED_BLINK_FAST    100   /* ms */
#define LED_BLINK_MEDIUM  500
#define LED_BLINK_SLOW    1000

volatile uint16_t blink_interval = LED_BLINK_FAST;
Led_TypeDef led_name = LED3;

int main(void)
{
  MPU_Config();
  CPU_CACHE_Enable();
  HAL_Init();
  SystemClock_Config();   /* 400 MHz via PLL1, HSE bypass */

  BSP_LED_Init(LED1);
  BSP_LED_Init(LED2);
  BSP_LED_Init(LED3);

  EXTI15_10_IRQHandler_Config();   /* PC.13 user button → EXTI */

  while (1)
  {
    BSP_LED_Toggle(led_name);
    HAL_Delay(blink_interval);
  }
}
```

Each press of the user button (PC.13, EXTI line 13) cycles through slow → medium → fast blink speeds and rotates the active LED. The EXTI callback includes a 200 ms debounce guard:

```c
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
  if (GPIO_Pin == GPIO_PIN_13)
  {
    uint32_t current_time = HAL_GetTick();
    if ((current_time - last_button_press) > DEBOUNCE_DELAY)
    {
      last_button_press = current_time;
      button_press_count++;
      if (button_press_count > 3) button_press_count = 1;

      switch (button_press_count)
      {
        case 1: blink_interval = LED_BLINK_MEDIUM; led_name = LED2; break;
        case 2: blink_interval = LED_BLINK_SLOW;   led_name = LED1; break;
        case 3: blink_interval = LED_BLINK_FAST;   led_name = LED3; break;
      }
    }
  }
}
```

<Tip>
Every example calls `MPU_Config()` and `CPU_CACHE_Enable()` before `HAL_Init()`. This pattern enables the Cortex-M7 L1 I-Cache and D-Cache and configures a catch-all MPU region to prevent speculative access to undefined memory. Copy this sequence into any new project.
</Tip>

## LL (low-layer) examples

`Projects/NUCLEO-H743ZI/Examples_LL/` provides the same peripheral concepts as `Examples/` but written against the LL API. LL drivers access hardware registers directly without the HAL abstraction overhead, which makes them suitable for latency-critical or memory-constrained code.

Six LL peripheral categories are available on NUCLEO-H743ZI:

```text
Examples_LL/
├── COMP/
├── DAC/
├── PWR/
├── RTC/
└── SPI/
```

<Note>
LL examples do not use `HAL_Init()` or `HAL_Delay()`. Initialization is done via `LL_*` functions. For example, clock configuration in `Templates_LL` calls `LL_RCC_HSE_Enable()`, `LL_RCC_PLL1_SetN()`, and `LL_RCC_SetSysClkSource()` directly instead of the HAL `RCC_OscInitTypeDef` structures.
</Note>

## Examples_MIX

`Projects/STM32H7B3I-DK/Examples_MIX/` contains a single `DMA2D/` project that combines HAL and LL calls within the same source file. This is useful when HAL covers most of a peripheral's setup but LL is needed for a time-critical path.
